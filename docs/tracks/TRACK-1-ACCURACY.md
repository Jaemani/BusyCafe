# Track 1 — 엔진 신뢰도 및 정확도

## 목적과 제품 경계

이 트랙은 카페 좌표에 **카페 주변 지역의 혼잡도**를 더 정확하게 매핑하고, 그 추정치를 어느 정도 믿을 수 있는지 검증 가능한 방식으로 표현한다. 결과는 매장 좌석 점유율, 대기 인원 또는 카페 내부 혼잡도를 뜻하지 않는다.

현재 엔진은 최근접 핫스팟 최대 3곳의 4단계 레벨을 역거리제곱 가중 평균한다. 신뢰도는 최근접 거리, 가장 최신 관측의 신선도, 이웃 수를 곱해 계산한다. 이 방식은 결정적이고 설명하기 쉽지만 다음 한계가 있다.

- 공식 핫스팟 폴리곤이 있는데도 대표점까지의 거리만 사용한다.
- 서로 면적과 평상시 인구가 다른 핫스팟의 4단계 레벨을 같은 척도로 취급한다.
- 가장 최신 관측 하나가 전체 기여 데이터의 신선도를 대표한다.
- 인접 핫스팟 간 불일치, API 결측·지연, 경계 효과를 신뢰도에 반영하지 않는다.
- 현재 신뢰도는 입력 조건의 품질 점수이며, 실제 적중 확률로 보정된 값이 아니다.
- 기존 Phase 6의 좌석 점유 관측은 매장 상태 라벨이다. 지역 혼잡도 엔진의 정답 라벨로 직접 사용하면 제품 정의와 평가 대상이 어긋난다.

## 목표 지표

모든 지표는 시간 순서로 분리한 검증셋에서 산출하고, 시간대·평일/주말·거리·커버리지 등급별로 함께 공개한다.

| 구분 | 지표 | 초기 승격 기준 |
|---|---|---|
| 지역 혼잡 순위 | 슬롯별 Spearman 상관 | 1차 ≥ 0.50, 안정화 목표 ≥ 0.60 |
| 지역 혼잡 레벨 | 한 단계 이내 정확도 | ≥ 80% |
| 고신뢰 결과 | 한 단계 이내 정확도 | ≥ 90% |
| 신뢰도 보정 | Expected Calibration Error | ≤ 0.08 |
| 데이터 신선도 | 정상 수집 시 25분 이내 관측 비율 | ≥ 99% |
| 안전성 | `uncovered` 영역의 혼잡도 표시 | 0건 |

표본 수가 충분해지기 전에는 신뢰도를 “정확할 확률”로 설명하지 않고, `근거 충분도` 또는 `데이터 신뢰도`로 노출한다. 확률형 의미는 calibration gate를 통과한 모델 버전에만 허용한다.

## 단계별 작업 패키지

### A1. 기준선과 재현 가능한 평가

- 현재 IDW를 `model_version=v1-idw-point`로 고정하고 입력 스냅샷, config, 결과를 재생할 수 있게 한다.
- 지역 혼잡 관측용 스키마와 평가 스크립트를 분리한다. 카페 좌석 관측은 별도 보조 데이터로 보존한다.
- 시간 기반 train/calibration/test 분할을 사용해 미래 데이터 누출을 차단한다.
- 전체 지표와 실패 사례를 `docs/EVAL_REPORT.md`에 누적 기록한다.

**Gate A:** 동일 입력 재실행 결과가 완전히 같고, v1 기준 지표와 구간별 표본 수가 기록되어야 한다.

### A2. 폴리곤 기반 공간 모델

- 공식 121개 장소 폴리곤의 topology, 겹침, 누락, 면적을 검증한다.
- `제안`: 카페가 폴리곤 내부에 있으면 해당 관측을 우선하고, 외부에서는 대표점이 아니라 폴리곤 경계까지의 거리를 사용한다.
- `제안`: 여러 폴리곤이 겹치면 단일 장소를 임의 선택하지 않고, 교차 상태를 evidence에 남긴 뒤 검증된 가중 규칙을 적용한다.
- 폴리곤 경계 안팎에서 점수가 급변하지 않는지 경계 fixture로 검증한다.

**Gate B:** v1 대비 전체 및 거리 구간별 핵심 지표가 악화되지 않고, 경계 구간 지표가 개선되어야 한다. 악화 시 v1을 유지한다.

### A3. 시간·연속 신호 정교화

- 각 contributor의 `observed_at`을 개별 가중치에 반영하고 오래된 관측은 제외 또는 감쇠한다.
- `AREA_PPLTN_MIN/MAX`는 구간 불확실성을 보존한 연속 신호로 저장한다.
- `제안`: 핫스팟별 요일×시간 기준선이 3주 이상 쌓인 뒤에만 “평소 대비” 신호를 계산한다. 서로 다른 장소의 원시 인구 수를 직접 비교하지 않는다.
- 시간 평활은 급증 탐지와 함께 평가해 실제 변화가 지워지지 않게 한다.
- 서울시 예측값은 실측 신호와 분리해 저장하고, 예측 시점별 정확도를 독립 평가한다.

**Gate C:** 최소 3주의 연속 스냅샷과 결측률 보고서가 있어야 한다. 시간대별 지표가 v1보다 개선되고 급증 구간 recall이 악화되지 않아야 한다.

### A4. 신뢰도 V2와 캘리브레이션

- 신뢰도를 공간 적합성, 관측 신선도, 이웃 합의도, 소스 건강도, 모델 검증 성능으로 분해해 저장한다.
- 기여 핫스팟 레벨이 크게 갈리거나 데이터 시각이 어긋나면 합의도를 낮춘다.
- 입력 품질 점수와 경험적으로 보정된 정확도 확률을 API와 UI에서 구분한다.
- 표본이 충분한 구간에 isotonic 또는 Platt calibration을 challenger로 평가한다.

**Gate D:** 최소 표본 수를 사전에 고정하고, 홀드아웃 ECE ≤ 0.08 및 고신뢰 그룹 정확도 ≥ 90%를 동시에 만족해야 확률형 신뢰도로 승격한다. 미달 구간은 low 또는 데이터 부족으로 강등한다.

### A5. 정답 데이터와 모델 개선 루프

- 지역 혼잡 정답은 동일 시각의 보행 밀도 관측, 공식 구역 관측 또는 검증된 제휴 신호로 수집한다.
- 사용자 제보를 받을 경우 “주변 거리 혼잡”과 “매장 자리 상황”을 별도 질문과 별도 라벨로 저장한다.
- 관측자 간 일치도와 라벨 품질을 측정하며, 중복·비정상 제보는 원본을 삭제하지 않고 평가 대상에서 제외한다.
- IDW, 폴리곤 모델, 시공간 challenger를 동일한 고정 검증셋에서 비교한다.

**Gate E:** 새 모델은 shadow 결과로 최소 2주 비교하고, 핵심 지표 개선과 회귀 없음이 확인된 뒤 버전을 승격한다. 모델 버전별 즉시 rollback 경로를 유지한다.

## 실험 운영 원칙

- 한 실험에서는 한 종류의 변경만 평가하고 데이터셋 버전, 파라미터, 코드 commit을 기록한다.
- 튜닝셋과 최종 검증셋을 분리하며, 최종 검증 결과를 본 뒤 같은 검증셋에 재튜닝하지 않는다.
- 전체 평균만으로 승격하지 않는다. 외곽·경계·주말·야간 중 하나라도 심각하게 악화되면 보류한다.
- 점수와 confidence 산출은 계속 결정적 순수 함수로 유지한다. 외부 API 호출은 계산 경로에 넣지 않는다.
- 실패한 실험도 가설, 결과, 폐기 이유를 남긴다.

## 하지 않을 것

- 지역 혼잡도를 카페 내부 좌석 점유율이나 대기 시간으로 표현하지 않는다.
- 근거 데이터가 없는 지역을 보간해 채우거나 high confidence로 표시하지 않는다.
- 검증 없이 복잡한 ML, 날씨·행사·교통 feature를 추가하지 않는다.
- 장소별 규모 차이를 보정하지 않은 원시 인구 수를 직접 비교하지 않는다.
- 평가 지표가 좋아 보이도록 관측 슬롯, 지역 또는 실패 표본을 사후 제외하지 않는다.
- 카페 POI 정확도와 혼잡도 엔진 정확도를 하나의 신뢰도 숫자로 합치지 않는다.

## 채택된 결정

- Phase 6의 주 평가 라벨을 지역 보행 혼잡으로 교체하고, 좌석 점유는 “한산할 카페 추천”의 간접 효용 지표로 별도 유지한다.
- 폴리곤 기반 모델을 v2의 첫 challenger로 삼고, ML은 A1~A4 데이터·검증 기반이 갖춰질 때까지 보류한다.
- 신뢰도 V2 구성요소와 `model_version`을 저장하되, 외부 API 필드 변경은 소비자 호환성을 보장하는 버전 정책과 함께 진행한다.
