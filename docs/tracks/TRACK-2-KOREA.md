# Track 2 — 서울에서 국내로 확대

## 목표와 원칙

서울 전용 구현을 도시별 데이터 공급자를 교체할 수 있는 국내 공통 플랫폼으로 확장한다. 지도와 카페 원장은 전국에서 동일한 제품 경험을 제공하되, 실시간 혼잡도는 출처·상업적 이용 권한·공간 해상도·갱신 주기를 검증한 지역에만 표시한다. 데이터가 없는 지역을 서울 모델로 추론하거나 주변 도시 값으로 채우지 않는다.

이 트랙은 혼잡도 엔진의 정확도 개선을 담당하지 않는다. Track 1이 정의한 점수·신뢰도 계약을 사용하고, 도시별 관측값을 그 계약에 맞게 정규화하는 데 집중한다.

## 채택된 설계 결정

1. **국내와 해외가 공유하는 범용 지역 모델을 먼저 만든다.** `RegionProfile`, `CrowdObservationProvider`, `PlaceCatalogProvider`, `CoverageProvider` 경계를 공통으로 사용하고, 서울 구현을 첫 번째 어댑터로 이전한다.
2. **도시 지원 상태를 `catalog_only`, `pilot`, `live`, `suspended`로 관리한다.** 카페 POI만 검증된 지역은 `catalog_only`, 내부 검증 중인 지역은 `pilot`, 모든 게이트를 통과한 지역만 `live`로 공개한다. 장애나 권리 변경 시에는 `suspended`로 즉시 내린다.
3. **혼잡도 표준값은 원본을 보존한 정규화 레코드로 전달한다.** 공급자 원본 레벨·인구 구간·geometry·관측 시각·수집 시각·예측·출처·라이선스 버전을 저장하고, 공통 1~4 레벨은 도시별 명시적 매핑으로만 만든다.
4. **카페 원장은 Overture Places를 재사용한다.** 전국 bbox 추출, 행정경계 판정, 카페 분류 품질, 릴리스 증분 비용과 상업적 사용 조건은 실제 릴리스로 다시 검증한다. `[VERIFY]`
5. **지도 렌더링은 MapLibre 기반을 유지한다.** OpenFreeMap의 국내 전역 품질, 운영 사용 정책, 트래픽 한도와 장애 대응은 배포 전 검증하고, 필요하면 동일 MapLibre 인터페이스 아래 자체 타일 또는 다른 공급자로 교체한다. `[VERIFY]`

## 범용 데이터 계약

`RegionProfile`은 `region_id`, `country_code`, `city_code`, 표시명·언어, timezone, 지원 geometry, 기본 viewport, 공급자 목록, 공개 상태와 모델 버전을 가진다. `region_id`는 공급자의 이름이 아니라 서비스 내부의 안정적인 식별자다.

`CrowdObservationProvider`는 도시별 원본을 다음 공통 필드로 정규화한다.

- `source_id`, `area_id`, `geometry`, `observed_at`, `fetched_at`
- 원본 혼잡 레벨과 인구 최소·최대값, 공통 레벨 1~4
- 예측 시계열, 원본 payload 참조, source/license version
- 데이터 품질 상태: `valid`, `partial`, `stale`, `invalid`

`CoverageProvider`는 특정 시각에 유효한 지원 geometry와 해상도, stale 기준을 반환한다. 프론트와 점수 엔진은 도시명을 기준으로 분기하지 않고 이 결과만 사용한다.

`PlaceCatalogProvider`는 Overture ID, 릴리스, 원본 sources, 좌표·분류·주소·연락처와 검증 상태를 반환한다. 공급자별 상세 링크는 현재 정책과 동일하게 검증된 canonical ID 또는 URL이 있을 때만 제공한다.

## 도시 온보딩과 검증 게이트

후보 도시의 공개 API, 지자체 센서, 공공데이터 또는 상업 유동인구 공급자를 조사한다. 국내 지자체별 실시간 혼잡 API의 실제 존재 여부, 제공 범위와 이용 조건은 도시마다 확인한다. `[VERIFY]` 통신사 기반 상업 데이터의 갱신 주기, 재배포 권한과 비용도 계약 전까지 확정하지 않는다. `[VERIFY]`

각 도시는 다음 게이트를 순서대로 통과해야 한다.

1. **권리 게이트**: 상업적 사용, 캐시·가공·재배포, 지도 중첩 표시, 보존 기간과 출처 표기를 문서로 확인한다.
2. **스키마 게이트**: 실제 응답 fixture로 geometry, timezone, 관측 시각, 결측·오류 응답, 레벨 의미와 예측 구조를 확정한다.
3. **품질 게이트**: 경계 유효성, 좌표계, 중복 area, 갱신 지연, 누락률과 이상치를 최소 7일 관측한다.
4. **운영 게이트**: quota, 호출 비용, 재시도 정책, 한 cycle 소요시간, 장애 시 stale 전환과 공급자 중단 절차를 부하 시험한다.
5. **제품 게이트**: 표본 현장 관측으로 Track 1의 정확도 기준을 통과하고, UI가 지원·경계·미지원 영역을 과장 없이 표현하는지 확인한다.

한 게이트라도 실패하면 `live`로 공개하지 않는다. 수정 가능한 실패는 `pilot`, 권리 또는 지속 가능성 실패는 `catalog_only`로 유지한다. 원본 응답과 검증 결과는 fixture 및 `docs/VERIFICATION.md`에 기록하고, 공급자 채택·중단은 ADR로 남긴다.

## 제품과 UI

- 지도 탐색과 카페 위치는 카탈로그가 검증된 국내 지역에서 제공한다.
- `live` 영역은 혼잡도 색상, 관측 시각, 원본 출처, 근거와 신뢰도를 표시한다.
- `pilot` 영역은 일반 사용자에게 실시간 혼잡도를 노출하지 않고 내부 검증 환경에서만 사용한다.
- `catalog_only` 영역은 카페만 표시하고 “이 지역은 아직 혼잡도 데이터가 없습니다”라고 안내한다.
- `suspended` 영역은 마지막 값을 현재처럼 표시하지 않으며, 일시 중단 상태와 마지막 정상 갱신 시각을 보여준다.
- 도시 경계가 아니라 실제 coverage geometry를 기준으로 `covered`, `fringe`, `uncovered`를 판정한다.
- 도시 검색·현재 위치 이동 시 지원 수준을 함께 안내하고, 데이터가 없는 지역에서도 지도와 카페 탐색은 계속 동작한다.

## 운영과 성능

- 도시별 worker queue와 동시성 상한을 두어 한 공급자의 지연이 다른 도시의 수집을 막지 않게 한다.
- API 요청 중 외부 공급자를 호출하지 않는다. 관측값과 POI는 PostgreSQL/PostGIS에 버전 단위로 적재하고 읽기 API는 materialized 결과만 제공한다.
- 카페·coverage·혼잡도는 벡터 타일 또는 줌 단계별 집계로 전달한다. 상세 정보는 사용자가 선택할 때 별도 조회한다.
- 캐시 키에는 `city_code`, 데이터 버전, 모델 버전과 타일 좌표를 포함한다. 도시 갱신 실패가 전체 캐시를 무효화하지 않게 한다.
- 도시별로 ingest 성공률, observation lag p50/p95, cycle duration, 결측 area 수, score materialize 시간, API p95와 타일 크기를 관측한다.
- 공급자 장애 시 마지막 정상 데이터는 보존하되 stale 임계치를 넘으면 혼잡도 레이어를 자동 중단한다. 카페 원장과 기본 지도는 계속 제공한다.

## 단계와 완료 기준

### K0 — 공통 경계 추출

서울 구현을 범용 계약의 기준 fixture로 고정하고, 도시별 설정과 공급자 어댑터를 코드 분기 없이 주입할 수 있게 한다. 기존 서울 점수와 API 응답에 회귀가 없어야 한다.

### K1 — 국내 후보 조사

후보 도시마다 데이터 출처·권리·공간 범위·갱신 주기·비용을 동일 양식으로 조사한다. `[VERIFY]` 게이트를 통과할 가능성이 가장 높은 1개 도시만 첫 pilot으로 선택하고, 선택 근거와 탈락 사유를 ADR에 남긴다.

### K2 — 단일 도시 pilot

실측 fixture, 최소 7일 운영 관측, Overture 표본 검수와 현장 골든셋을 확보한다. 누락·장애 상황에서도 서울 운영과 카페 탐색이 영향을 받지 않아야 한다.

### K3 — 국내 반복 확장

새 도시 추가가 공통 스키마와 설정, 공급자 어댑터 구현만으로 가능해야 한다. 프론트 도시별 조건문과 DB 스키마 변경은 허용하지 않으며, 필요하다면 범용 계약 자체의 ADR을 먼저 작성한다.

국내 `live` 도시의 최소 성공 기준은 다음과 같다.

- 데이터 권리 게이트를 문서 증거로 통과한다.
- 7일 ingest 성공률 99% 이상, observation lag p95가 공급자 약속 주기의 2배 이하이다.
- 유효 coverage의 95% 이상 area가 각 정상 cycle에 갱신된다.
- Track 1의 현장 정확도·신뢰도 calibration 합격선을 통과한다.
- cached 지도 타일 p95 50ms 이하, 화면당 지도 데이터 전송량 200KB 이하를 목표로 한다.
- 신규 도시 장애가 서울 API와 worker SLA에 영향을 주지 않는다.

## 명시적 비범위

- 검증되지 않은 전국 단일 혼잡도 모델로 빈 지역을 채우지 않는다.
- 지도·통신사·플랫폼 데이터를 스크레이핑하지 않는다.
- 공급자마다 의미가 다른 혼잡 레벨을 이름만 같다는 이유로 동일하게 취급하지 않는다.
- 전국 POI를 한 번에 활성화하지 않는다. 릴리스별 지역 표본 검수와 원자적 승격을 거친다.
