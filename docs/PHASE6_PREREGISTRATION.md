# Phase 6 사전 등록 분석 계획 (v1 기준선 · v1/v2 paired 판정)

- 상태: **사전 등록 — 첫 현장 관측 전 확정**
- 등록일: 2026-07-12 (Asia/Seoul)
- 등록 시점 코드 commit: `4480f15` (divergence 목록 생성 commit: `7899d6b`)
- 입력 고정물: [`SHADOW_REPORT.md`](SHADOW_REPORT.md)의 결정적 divergence 정렬(절대
  score 차이 내림차순, cafe ID 오름차순),
  [`PHASE6_CANDIDATES_2026-07-12.csv`](PHASE6_CANDIDATES_2026-07-12.csv) 24곳
- 변경 규칙: 첫 관측 이후의 계획 변경은 ADR로 기록하고 결과 보고서에 **post-hoc**로
  표기한다. 관측 결과를 본 뒤 지표·표본·검정을 바꾸지 않는다.

## 1. 왜 사전 등록인가

v1/v2는 4,933곳 중 score 137곳(3.6%), 표시 level 29곳(0.75%)만 다르다. 무작위에
가까운 primary 표본 24곳에 divergent 카페가 포함될 기대값은 약 1곳이므로, **primary
기준선만으로는 v1/v2 우열 판정이 통계적으로 불가능하다.** divergence 목록은 관측
결과를 보기 전에 결정적으로 생성됐으므로, 지금 분석 계획을 고정하면 사후 선택 편향
없이 층화 표본으로 사용할 수 있다. 이 문서가 그 고정이다.

## 2. 관측 프로토콜 (모든 층 공통)

- 이중 라벨: **지역 보행 혼잡**(모델 정답 라벨)과 **매장 좌석 점유**(제품 효용 지표)를
  별도 기록한다. 모델 게이트 판정에는 보행 혼잡 라벨만 사용한다.
- 라벨 척도는 PLAN §5 Phase 6 기준표(1~4)를 사용하고, 이중 관측자 입력 계약을 따른다.
- 관측 시각을 분 단위로 기록하고, historical no-leak evaluator로 동시각 snapshot
  기준 v1/v2 예측을 재생한다. 예측을 관측 후에 재계산하되 입력은 관측 시각 이전
  snapshot만 사용한다.
- 제외 규칙: 폐업·이전·접근 불가 카페는 `exclusion_reason`을 기록하고, 대체는 각 층의
  사전 정의된 정렬 순서에서 다음 후보로만 한다. 관측 후 성적 사유의 제외는 금지한다.

## 3. 층(strata) 정의

### S1 — Primary 기준선 (기존 계획 유지)

- 표본: `PHASE6_CANDIDATES_2026-07-12.csv` 24곳 × 동네별 3슬롯.
- 목적: **v1 절대 기준선과 제품 가설 검증.** Track 1 승격 기준표를 따른다
  (슬롯별 Spearman 1차 ≥ 0.50, adjacent accuracy ≥ 80%, 거리 구간 분해).
- 이 층은 v1/v2 비교에 사용하지 않는다. 검출력 부족을 사전에 인정한다.

### S2 — Divergence 층 (v1/v2 판정 전용)

- 표본: SHADOW_REPORT 감사 목록 정렬 순서대로 **표시 level이 변경된 카페**를 위에서부터
  채워 20곳(운영 상태 검수 후 최소 16곳). 검수는 영업 여부·접근성만 보며 혼잡 상태를
  근거로 제외하지 않는다. 각 2슬롯 → 목표 32~40 관측.
- 가설 H1: 보행 혼잡 관측 레벨 `obs`에 대해 `|v2−obs| < |v1−obs|`인 관측이 절반을
  초과한다.
- 검정: 동점(`|v2−obs| = |v1−obs|`) 제외 exact sign test, 양측 α=0.05. 동점 비율과
  비동점 표본 수를 함께 보고한다.
- **비대칭 guardrail**: v2가 v1보다 낮게 예측한 관측에서 `obs − v2 ≥ 2` 레벨인 사례가
  1건이라도 나오면 승격 불가. 근거: divergence 상위가 v2 하향 방향에 집중돼 있고,
  "붐비는 곳을 한산하다고 표시"는 이 제품의 최악 실패 양식이다.

### S3 — Coverage 확장 층 (report-only)

- 표본: v1 `uncovered` → v2 `covered/fringe`로 확장된 266곳에서 경계거리 층화로
  8곳 × 2슬롯.
- 성격: 게이트가 아니라 보고서다. 확장 구간 추정의 계통 오류 여부를 기록한다.

## 4. 승격 결정 규칙

v2 승격은 두 결정으로 **분리**한다.

1. **기존 covered/fringe 구간의 스코어 모델 교체**: S1 게이트 통과 **그리고** S2에서
   v2의 유의한 우위(sign test) **그리고** 비대칭 guardrail 0건일 때만 검토한다.
   S2가 동등(비유의)이면 v1을 유지하고 v2는 shadow로 지속한다 — 차이 나는 지점에서
   우열을 가릴 수 없으면 더 단순한 모델이 남는다.
2. **coverage 확장 활성화**: 1이 승인되고 S3 결과의 HUMAN 리뷰를 통과한 뒤 별도로
   결정한다. 확장 없이 스코어 교체만 배포할 수 있다.

### 기존 게이트 문구 개정 (SHADOW_REPORT '다음 gate' 4항을 대체)

- "coverage loss 0"은 모델 품질 증거가 아니라 **제품 연속성 제약**(사용자에게 보이던
  카페가 조용히 사라지지 않게 하는 배포 제약)으로 재분류한다. 정당한 사유가 검증되면
  coverage 축소도 허용한다. coverage 단조 증가를 품질 기준으로 삼지 않는다.
- v2 판정에 v1 confidence tier 임계값을 적용하지 않는다. v2의 coverage/confidence는
  같은 단어라도 다른 분포다(내부 카페 coverage_factor=1.0). Confidence V2는 ECE를
  측정할 수 있을 때까지 서술적 구성요소로만 사용하고, v2 confidence tier 임계값은
  승격 전 재유도한다.
