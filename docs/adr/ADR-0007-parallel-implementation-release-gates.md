# ADR-0007: 구현 병렬화와 공개·승격 게이트를 분리한다

- 날짜: 2026-07-12
- 상태: Accepted
- 결정자: 사용자, Codex
- 관련 문서: `docs/PLAN.md`, `docs/ROADMAP.md`, `docs/audits/2026-07-12-external-review-response.md`

## 맥락

기존 계획은 한 Phase의 모든 DoD를 통과해야 다음 Phase로 넘어간다고 규정했다. 실제로는
독립적인 코드와 fixture 작업을 병렬로 구현하면서도 이전 Phase의 HUMAN 검수 때문에
공개 승격만 대기했다. 문서 규칙과 실제 관행의 차이가 감사 가능성을 떨어뜨렸다.

## 결정

- 독립적이고 되돌릴 수 있는 구현·테스트·조사는 Phase 간 병렬로 진행할 수 있다.
- 사용자 공개, 기본 경로 전환, 데이터 source 승격과 완료 선언은 선행 게이트를 모두
  통과해야 한다.
- 게이트 대기 결과는 production 기본값으로 활성화하지 않으며 상태를 명시한다.
- HUMAN 검수나 외부 credential이 필요한 항목은 블로커로 기록하되 독립 작업 전체를
  중단시키지 않는다.
- migration은 실제 적용 테스트와 대상 DB 적용을 통과하기 전 실행 서비스가 새 schema를
  로드하지 못하게 한다.

## 결과

Phase 상태는 `구현 전`, `구현 완료·게이트 대기`, `승격 완료`를 구분한다. “병렬 구현”을
규칙 위반의 예외로 취급하지 않고, 공개와 완료 판정에만 엄격한 순서를 적용한다.
