# ADR-0006: 세 개 제품 트랙과 유니버설 혼잡 데이터 계약

- 날짜: 2026-07-11
- 상태: Partially superseded by ADR-0010 (contract-first 순서만 대체)
- 결정자: 사용자, Codex
- 관련 문서: `docs/ROADMAP.md`, `docs/tracks/`

## 맥락

서울 카페 지도에서 엔진 정확도 개선, 국내 확대, 해외 확대를 동시에 검토해야 한다.
지역마다 혼잡 데이터 형식과 라이선스가 달라 도시별 분기 코드를 추가하면 모델 검증과
운영이 빠르게 분산된다. 카페 지도와 지역 밀집도 지도도 같은 관측 데이터를 공유해야 한다.

## 결정

제품 작업을 다음 세 트랙으로 분리한다.

1. 엔진 신뢰도 및 정확도
2. 서울에서 국내로 확대
3. 해외로 확대

세 트랙은 장소 원장, 혼잡 관측, 커버리지, 지역 설정으로 구성된 공통 데이터 계약을
사용한다. 성능, 실시간 저장소, 벡터 타일과 관측성은 공통 플랫폼 레인에서 한 번만
구현한다.

> 2026-07-13: 위 계약을 실측 전에 먼저 확정하는 순서는 ADR-0010이 대체한다. 현재
> `universal_contracts.py`는 experimental seam inventory이며, 두 번째 provider fixture에서
> 확인된 최소 교집합만 런타임 계약으로 승격한다. 세 트랙과 미지원 영역을 채우지 않는
> 원칙은 유지한다.

사용자 지도는 카페 찾기, 지역 밀집도, 데이터 커버리지 모드를 제공한다. 혼잡 데이터가
없는 지역에서도 장소 원장은 표시할 수 있지만 혼잡도를 추정해서 채우지 않는다.

## 근거

- Overture Places와 MapLibre 기반 지도는 지역에 독립적이다.
- 실시간 혼잡 데이터는 공급자와 도시마다 갱신 주기, 공간 단위와 라이선스가 다르다.
- 동일 observation 계약을 사용하면 서울에서 검증한 엔진, API와 UI를 국내·해외에
  재사용할 수 있다.
- 별도 커버리지 모드는 데이터 공백을 숨기지 않는 제품 원칙을 지킨다.

## 결과와 재검토 조건

- 도시별 구현은 어댑터와 설정으로 제한한다.
- 도시 간 원본 혼잡 수치를 직접 비교하지 않고 도시별 상대 기준으로 정규화한다.
- 공급자 라이선스나 데이터 품질이 공통 계약으로 표현되지 않을 때만 계약 확장을 검토한다.
- 매장 점유율 데이터가 확보되면 지역 혼잡도와 별도 모델·필드로 관리한다.
