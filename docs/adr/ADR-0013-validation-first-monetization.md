# ADR-0013: 검증 전에는 광고하지 않고 후원부터 제한적으로 도입한다

- 상태: Accepted
- 날짜: 2026-07-15
- 관련 문서: `docs/PRODUCT_METRICS.md`, `frontend/public/privacy.html`

## 맥락

Busy Cafe의 핵심 가치는 지도 화면 면적과 혼잡 추정에 대한 신뢰다. 초기 트래픽에서
디스플레이 광고는 수익이 작지만 모바일 지도를 가리고 성능, 개인정보 동의와 외부 추적기
리스크를 즉시 만든다. 제품 정확도와 반복 사용도 아직 현장 gate를 통과하지 않았다.

반면 자발적 후원과 명확히 분리된 지역 스폰서는 제품 결과를 바꾸지 않는 조건에서 운영비를
보조할 수 있다.

## 결정

1. 현재 공개 베타에는 디스플레이 광고를 넣지 않는다.
2. 후원 계정 URL이 준비되면 개인정보·운영 안내 또는 정보 영역 하단에 한 개의 텍스트
   링크만 둔다. 팝업, interstitial, 지도 위 sticky 배너와 반복 요청은 사용하지 않는다.
3. 지역 스폰서는 `docs/PRODUCT_METRICS.md`의 정확도·운영·사용량 gate를 통과한 뒤 별도
   슬롯으로만 시험한다. `스폰서`임을 명확히 표시한다.
4. 후원·스폰서 여부는 혼잡 점수, 마커 색, 검색 결과, 순위, evidence와 coverage에 어떤
   영향도 주지 않는다.
5. 개인화 광고와 위치 기반 광고 추적기는 도입하지 않는다. 장기적으로 광고를 검토해도
   문맥 기반이어야 하며 별도 개인정보·성능·법적 검토를 먼저 통과해야 한다.
6. 스폰서 pilot은 지도 마커·혼잡 결과와 분리된 슬롯에서 한 화면 최대 한 개만 노출한다.
   `광고` 또는 `스폰서`를 명시하고 외부 링크에 `rel="sponsored noopener noreferrer"`를
   사용한다. 팝업, 자동 이동과 정밀 위치 기반 targeting은 금지한다.
7. 스폰서 문구는 `지금 한산함`, `자리 있음`, `대기 없음`처럼 Busy Cafe가 검증하지 않은
   혼잡·영업 주장을 할 수 없다. stale/uncovered 결과에 붙여 제품 근거처럼 보이게 하지
   않으며, 목적지 host·문구·광고주를 사전 검수하고 신고 시 내릴 절차를 둔다.
8. pilot 전에 `sponsor_impression`과 `sponsor_click`의 저카디널리티 schema, 노출 상한,
   bot·중복 제외와 목적지 allowlist를 별도 테스트한다. custom event가 비활성인 동안
   성과형 과금이나 클릭 성과 주장을 하지 않는다.

## 단계별 gate

- 후원 링크: analytics가 실제 수집되고 개인정보 안내가 공개된 뒤 운영자 URL 확인
- 스폰서 pilot: Phase 6와 freshness SLO 합격, 최근 30일 production pageview 10,000건과
  `external_map_click` 1,000건을 모두 관측. custom event 미지원 중에는 판정 보류
- 디스플레이 광고 재검토: 최근 30일 production pageview 200,000건과 안정적인 사용 흐름을
  관측한 뒤에도 지도 UX·LCP/INP·privacy 비용보다 기대 수익이 큰 경우만

수치는 초기 운영 가설이며 실제 분포를 기록한 뒤 변경할 수 있다. 다만 결과 불변과 광고
표시 의무는 사용량과 무관한 고정 정책이다.

후원 링크는 광고가 아닌 자발적 결제로 취급하되 Busy Cafe가 결제정보를 직접 받지 않는
외부 제공자 링크만 사용한다. 제공자명, 수수료·환불·개인정보 처리 주체를 링크 주변 또는
제공자 화면에서 확인할 수 있어야 하며 운영자 URL 검증 전에는 빈 버튼을 노출하지 않는다.

## 결과

초기 수익은 제한되지만 제품 가설과 신뢰를 먼저 측정할 수 있다. 후원 계정 생성과 URL
확정은 `[HUMAN]`이며, 확인 전에는 빈 링크나 임의 계정을 제품에 노출하지 않는다.
