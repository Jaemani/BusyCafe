# ADR-0009: 생활인구 베이스라인 단위로 250m 격자를 채택한다

- 날짜: 2026-07-12
- 상태: Accepted
- 결정자: 사용자, Claude
- 관련 문서: [`docs/research/2026-07-12-living-population.md`](../research/2026-07-12-living-population.md),
  `docs/tracks/TRACK-1-ACCURACY.md`, `docs/VERIFICATION.md` 2026-07-12 항목

## 맥락

밀집도 엔진 고도화 방향은 2층 구조다: 지점별 시간대 베이스라인 × 실시간 이상치 비율
전파. 베이스라인 층과 "지연된 정답" 백테스트 모두 서울 생활인구 데이터가 필요하다.
그런데 집계구 단위 데이터셋(OA-14979 등 7종)은 공식 공지 기준 **2026-07-31 18:00
서비스 종료**되고, KT 원천 데이터가 국가표준 250m 격자로 전환된다. 결정을 미루면
파서·fixture를 3주 뒤 폐기되는 단위에 맞추게 된다.

## 고려한 선택지

1. **집계구(OA-14979)** — 2017년부터 깊은 이력, 그러나 신규 공급이 19일 뒤 종료되고
   경계가 불규칙(2016년 기준 19,153개)이라 공간 조인이 복잡하다.
2. **행정동(OA-14991)** — 계속 제공되지만 해상도가 낮아(서울 ~426개) 카페 단위
   베이스라인에 부족하다.
3. **250m 격자(OA-22784/85/86)** — 국가표준 단위로 신규 공급이 지속되고, 월별 이력이
   2023-01부터 존재하며, 정규 격자라 핫스팟 폴리곤·카페 좌표와의 조인이 단순하고,
   Track 2 국내 확장과 단위가 호환된다.

## 결정

생활인구 파이프라인의 목표 단위는 **250m 격자**로 한다. 아카이브된 집계구 데이터는
2023년 이전 심층 이력이 필요할 때만 보조 백테스트 소스로 검토한다(아카이브 URL과
보존 기한은 `[VERIFY]`).

## 근거 (2026-07-12 실측)

- Open API 서비스명 `Se250MSpopLocalResd` 확인. 요청 인자에 기준일(YMD)이 필요하다.
- 기존 `SEOUL_API_KEY`로 호출 성공: **XML은 INFO-000 정상, JSON은 동일 요청이
  ERROR-500** — 포털 측 결함으로 보이며 ingest는 XML을 기본으로 한다(재검토 조건).
- 20260708 자료 `list_total_count=253,751`(≈10,573셀 × 24시간), `TT` 필드로 시간대별
  제공 확인.
- `CELL_ID`는 국가지점번호 형식(예: `다사52505325`) — geometry를 ID에서 산술 파생할
  가능성이 높다(`[VERIFY]` 후 좌표 조인 방식 확정).
- 데이터 특성: 개인정보 비식별화로 성·연령 구간이 3명 이하이면 `*` 마스킹,
  `H_DNG_CD`에 후행 공백 존재(파서는 strip 필수), Sheet/OpenAPI는 4일 전 당일자료만
  제공(대량 이력은 파일 다운로드).
- 라이선스: 공공누리 1유형(출처표시, 상업적 이용·변경 가능). attribution 페이지 반영은
  `[HUMAN]` 확인 대상.
- 원본 fixture: `backend/fixtures/se250m_spop_local_resd_sample.xml`
  (SHA-256 `efa7423dafa4017376d9b2b7a5feaeae4fd1dbcf656c216b609d8a892ec401fd`).

## 결과와 재검토 조건

- 파서·스키마 확정은 파일 다운로드 원본(월별 ZIP)까지 확보한 뒤 진행한다
  (INC-2026-001 원칙: 원본 먼저, 파서 나중).
- JSON 타입이 복구되면 XML 기본 결정을 재검토한다.
- `CELL_ID` → geometry 변환이 검증 실패하면 격자 경계 파일 확보로 대체한다.
- 생활인구는 존재 인구이지 보행 흐름이 아니다 — citydata 혼잡 라벨과의 상관 검증이
  베이스라인 활용의 선행 게이트다(Track 1).
