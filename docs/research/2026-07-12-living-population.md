### 2.2 접근 방법

- **형식**: 전 데이터셋이 파일(ZIP, 내부 CSV 안내) + Open API를 제공한다. 인코딩은
  명시되지 않아 `[VERIFY]`.
- **파일 크기**([OA-14979](https://data.seoul.go.kr/dataList/OA-14979/F/1/datasetView.do)
  확인): 집계구 내국인 일별 약 38~41MB, 월별 약 530MB~1.45GB(122파일, 2017-01~2026-06).
  집계구 장기체류외국인 일별 약 4.2~4.5MB, 월별 약 70~150MB. 250m 격자 내국인 일별 약
  14.5~14.7MB, 월별 약 380~600MB.
- **공개 지연**: 여러 페이지가 "Sheet와 OpenAPI는 4일전 당일자료만 제공"이라고 안내했다
  (FAQ 일부는 "5일전" — 4~5일 지연으로 본다). 전체 CSV 다운로드가 더 최신 범위를
  포함하는지는 실측 전까지 `[VERIFY]`.
- **행정동 단위**: [OA-14991](https://data.seoul.go.kr/dataList/OA-14991/S/1/datasetView.do)은
  Sheet/API가 최근 2개월만 제공(파일 다운로드는 2017년 상반기부터).
- **API 인증키**: [Open API 이용안내](https://data.seoul.go.kr/together/guide/useGuide.do)에서
  "인증키를 먼저 발급받아야 한다"를 확인했다. 신청 종류는 "일반 인증키"뿐이라
  `PLAN.md`의 기존 `SEOUL_API_KEY`가 커버할 가능성이 높지만 실호출 전엔 확정 불가.
  **`[HUMAN]`** 실호출 검증 필요.
- Sheet/API 미리보기는 최대 1,000건까지만 노출된다.

### 2.3 필드 스키마 — `[VERIFY]`

상세 페이지의 Sheet/Open API 탭은 동적 렌더링이라 컬럼명을 직접 확인하지 못했다. 공식
페이지에서 간접 확인한 것: 성별·연령대별로 분해된 필드 존재, 3명 이하 값은 `*` 마스킹,
소수점 5자리까지 존재하며 세부 합이 전체 합계와 불일치할 수 있음(모두
[OA-14979 파일 탭](https://data.seoul.go.kr/dataList/OA-14979/F/1/datasetView.do)과
[안내 페이지](https://data.seoul.go.kr/dataVisual/seoul/seoulLivingPopulation.do)에서 확인).
비공식 블로그에서 자치구 일별 API(`SPOP_DAILYSUM_JACHI`) 필드명으로 `STDR_DE_ID`(기준일),
`SIGNGU_CODE_SE`(자치구코드), `TOT_LVPOP_CO`(총생활인구수)를 봤다는 정보가 있으나 공식
출처가 아니라 `[VERIFY]`다. 정식 레코드 레이아웃은 실제 CSV/API 응답 확보 후 확정한다.

### 2.4 집계구 경계 geometry

[안내 페이지](https://data.seoul.go.kr/dataVisual/seoul/seoulLivingPopulation.do)에서
data.seoul.go.kr이 "통계지역경계(집계구.shp, 2016)"와 데이터 설명서를 자체 파일로 제공함을
확인했다. 개별 다운로드 링크(파일명)까지는 클릭 확인하지 못했다 `[VERIFY]`.

대안으로 [SGIS 자료제공 안내](https://sgis.mods.go.kr/view/pss/dataProvdIntrcn)(구
sgis.kostat.go.kr은 이 주소로 302 리다이렉트됨)를 직접 fetch했다: 회원가입·로그인 **필수**,
자료신청 1단계 입력 → 이용약관 동의 → 자료 선택 → 신청 완료 → 다운로드의 4단계 절차.
파일 다운로드 자체에 별도 API 키가 필요하다는 언급은 없었다. 좌표계는 "UTM-K(GRS80
타원체)"로 명시(EPSG 5179 여부는 본문에 숫자로 없어 `[VERIFY]`). 집계구별 5명 미만 통계는
서비스 제외.

결론: 서울시 자체 shapefile을 1차 후보로, 안 되거나 최신 경계가 필요하면 SGIS 신청을
백업으로 쓴다.

### 2.5 데이터 의미상 주의점

[안내 페이지](https://data.seoul.go.kr/dataVisual/seoul/seoulLivingPopulation.do) 정의:
생활인구는 "서울시와 KT가 공공빅데이터와 통신데이터를 이용하여 추계한 서울의 특정지역,
특정시점에 존재하는 모든 인구"다. 같은 페이지 비교표는 생활인구·등록인구·인구총조사를
공간·시간 범위별로 구분한다 — 생활인구는 **체류(존재) 인구**이며 거주자·재택자·실내
근무자를 포함한다. 이는 카페 앞 "보행 흐름"(`TRACK-1-ACCURACY.md` 주 정답 라벨)과 다른
개념이다. citydata_ppltn의 `AREA_PPLTN_MIN/MAX`도 기지국 신호 기반 존재인구라 개념상
가깝지만 공간 단위(핫스팟 폴리곤 vs 집계구/격자)와 산출 방식이 달라, 두 신호의 상관관계는
**확인해야 할 가설이지 전제가 아니다** — `TRACK-1-ACCURACY.md`의 "정답 없는 구조 비교를
정확도 증거로 쓰지 않는다" 원칙과 같은 태도로 다룬다.

## 3. 라이선스 — `[HUMAN]` 최종 판단 필요

확인한 모든 데이터셋 페이지(OA-14979, OA-14978, OA-14991, OA-22784)에 동일 표기:
**"공공누리 1유형: 출처표시 (상업적 이용 및 변경 가능)"**, 제3저작권자 "없음".

[열린데이터광장 이용안내](https://data.seoul.go.kr/etc/openInfo.do)에서 공통 정책도
확인했다: 저작권법 제24조의2 근거로 공공누리 표시 저작물은 표시 조건 범위에서 자유이용
가능, "공공데이터는 저작권자 표시 조건만으로 자유이용을 허락함을 원칙으로 한다", 단
데이터셋별 이용조건이 다를 수 있어 개별 메타정보 확인 필요.

SGIS 경계 자료는 신청 시 "이용약관 동의"가 필요하다는 것만 확인했고, 구체적 공공누리
유형이나 상업적 이용 허용 문구는 fetch한 본문에 없어 `[VERIFY]`.

**`[HUMAN]`**: 공공누리 1유형은 서비스 목적에 부합해 보이나 (1) 종료 예정 집계구 데이터에
계속 의존할지, (2) SGIS 경계 데이터 정확한 이용조건, (3) 출처표시 노출 방식은 사람이
최종 확정한다.
