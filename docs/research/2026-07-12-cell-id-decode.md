# CELL_ID(국가 격자) 디코드 규칙과 표본 검증

- 상태: 표본 검증 완료(authority 대조는 `[VERIFY]`)
- 작성일: 2026-07-12
- 관련: [ADR-0009](../adr/ADR-0009-living-population-250m-grid.md),
  `backend/app/ingest/national_grid.py`, `backend/tests/test_national_grid.py`

## 1. 디코드 규칙

생활인구 250m 격자 피드의 `CELL_ID`(예: `다사52505325`)는 국가지점번호 계열 인코딩으로
해석한다. 구현은 `backend/app/ingest/national_grid.py`.

- 한글 2자 = 100km × 100km 구역. 기준 모서리 `가가 0000 0000`은 UTM-K(EPSG:5179)
  E 700,000 / N 1,300,000. 첫 자는 동쪽(가~사), 둘째 자는 북쪽(가~아) 카운트.
  서울은 `다사`(E +200km, N +600km).
- 숫자 8자리 = 4+4 (동쪽, 북쪽 오프셋), 단위 10m, 셀의 남서 모서리 주소.
  250m 격자이므로 값은 25의 배수여야 하며 아니면 거부한다.
- EPSG:5179 → WGS84 역변환은 Snyder(USGS PP 1395) 표준 TM 역급수를 직접 구현
  (pyproj 의존성 없음). Korea 2000(GRS80)과 WGS84의 datum 차이는 본 용도에서 무시
  가능 수준이라 보정하지 않는다.
- 규칙 출처는 국가지점번호 정의(도로명주소법 시행령)를 따른 것으로, 공식 고시 문서
  원문 URL 확인은 `[VERIFY]`다. 아래 실측 검증이 현재의 근거다.

## 2. 표본 검증 (2026-07-12, 실데이터 1000행)

입력: `Se250MSpopLocalResd` 20260708 실응답 1000행(고유 셀 817개, 행정동 48개).
검증 스크립트는 scratchpad에서 실행(일회성, 저장소 미포함).

| 검증 | 결과 |
|---|---|
| 서울 bbox 포함(±0.01°) | **817/817 통과** |
| 동일 행정동 셀 군집(최대 쌍거리) | 중앙값 1,344m, 최악 3,603m(103셀 대형 동) — 전부 한 행정동 크기로 타당 |
| 자치구 스팟체크 | `11110`(종로구) 389셀 centroid (37.5899, 126.9806) — 종로구 내부 PASS. 표본에 종로·용산만 존재해 2개 구 확인 |
| 왕복 투영 일관성(역→순방향) | 최대 오차 0.000011m |
| 인접 셀 간격 | 250.56m (위도 수렴 효과 포함, 250m 격자 확정) |
| 고정 앵커 | `다사52505325`(H_DNG_CD 11110515, 종로구) → (37.578539, 126.963463) — 회귀 테스트로 고정 |

## 3. 한계와 남은 검증

- 이 검증은 **표본 기반 자기일관성 + 행정구역 정합**이다. 공식 250m 격자 경계
  파일(통계청/SGIS)과의 전수 대조는 `[VERIFY]`로 남는다 — 파이프라인에서 geometry가
  점수에 영향을 주기 시작하기 전에 수행한다.
- 표본이 종로·용산 구간(정렬 첫 1000행)이라 서울 외곽 셀은 bbox 검증만 거쳤다.
  전일 데이터(253,751행) 전수 디코드는 파일 원본 확보 후 수행한다.
- 왕복 오차는 급수 구현의 자기일관성만 보증한다. 절대 정확도의 근거는 행정구역
  정합(종로구 389셀)과 인접 간격이다.
